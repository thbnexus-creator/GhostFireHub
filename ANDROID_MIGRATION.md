# GhostFireHub 2.0 — Android Native Migration & Handoff Specification

## Executive Overview
This document specifies the database schemas, security rules, storage hierarchy, state contracts, and Kotlin/Compose data models required to port the GhostFireHub 2.0 Web Application into a native Android app powered by Firebase SDKs (Firebase Auth, Cloud Firestore, Firebase Storage, and Firebase Cloud Messaging).

---

## 1. Firebase Cloud Firestore Architecture & Schemas

### Document Collections

#### Collection: `users`
- **Document ID**: `{uid}` (Firebase Auth User ID)
- **Fields**:
  - `uid` (`String`): Auth UID
  - `email` (`String`): Primary account email address
  - `displayName` (`String`): Gamer alias
  - `role` (`String`): `'Gamer'` | `'Vendor'` | `'Administrator'` | `'SuperAdmin'`
  - `isPremium` (`Boolean`): Premium VIP subscription toggle
  - `membershipTier` (`String`): `'Gold'` | `'Platinum'` | `'Diamond'`
  - `ghostPoints` (`Number`): Platform reward point balance
  - `deviceBrand` (`String`): Device brand (e.g. Xiaomi, Tecno, Infinix)
  - `deviceModel` (`String`): Specific hardware model name
  - `referralCode` (`String`): Unique invite code
  - `referralCount` (`Number`): Successful user referrals count
  - `fcmToken` (`String`, Optional): Active FCM Registration Token
  - `dataSharingConsent` (`Boolean`): Anonymized telemetry opt-in flag
  - `createdAt` (`String`, ISO 8601): Registration timestamp
  - `updatedAt` (`String`, ISO 8601): Last update timestamp

#### Collection: `marketplace`
- **Document ID**: `{productId}` (Auto-generated UUID or Firestore ID)
- **Fields**:
  - `id` (`String`): Product unique identifier
  - `name` (`String`): Product name
  - `category` (`String`): `'Accounts'` | `'Coaching'` | `'Skins'` | `'HUD Layouts'` | `'Config Files'`
  - `price` (`Number`): Price in points or local currency
  - `description` (`String`): Full product description
  - `image` (`String`): Image URL stored in Firebase Storage (`marketplace/`)
  - `vendorId` (`String`): UID of the publishing vendor
  - `vendorEmail` (`String`): Email of the publishing vendor
  - `status` (`String`): Enum `'Draft'` | `'Submitted'` | `'Pending Admin'` | `'Approved'` | `'Rejected'` | `'Needs Evidence'`
  - `riskLevel` (`String`): `'Low'` | `'Medium'` | `'High'`
  - `riskScore` (`Number`): Integer score (0–100) generated by GhostCore AI
  - `aiReviewNotes` (`String`): Automated policy compliance notes
  - `aiReviewLogs` (`Array<String>`): Detailed array of AI review execution steps
  - `evidenceRequested` (`Boolean`): Verification evidence flag
  - `evidenceUrl` (`String`, Optional): Evidence asset URL in Storage (`verification/`)
  - `hidden` (`Boolean`): Visibility toggle for vendor
  - `createdAt` (`String`, ISO 8601): Submission timestamp
  - `approvalDate` (`String`, Optional, ISO 8601): Timestamp when Super Admin approved
  - `approvedBy` (`String`, Optional): Super Admin email who approved listing

#### Collection: `vendor_applications`
- **Document ID**: `{applicationId}`
- **Fields**:
  - `id` (`String`): Application ID
  - `username` (`String`): Applicant username
  - `email` (`String`): Applicant email
  - `brandName` (`String`): Vendor store/brand name
  - `experienceDescription` (`String`): Background and experience statement
  - `status` (`String`): Enum `'Pending'` | `'Approved'` | `'Rejected'`
  - `vendorKey` (`String`, Optional): Unique corporate key assigned upon approval
  - `createdAt` (`String`, ISO 8601): Timestamp of submission

#### Collection: `settings`
- **Document ID**: `global_config`
- **Fields**:
  - `maintenance_mode` (`Boolean`): Global site maintenance toggle
  - `allow_registrations` (`Boolean`): Registration enablement flag
  - `giveaway_submission_open` (`Boolean`): Giveaway entry submission toggle
  - `min_app_version` (`String`): Minimum required app version for force update
  - `subscriptions` (`Map`): Dynamic pricing and features per tier (`Gold`, `Platinum`, `Diamond`)
  - `payment_methods` (`Map`): Active payment gateways (`bank_transfer`, `crypto_wallet`, `telegram`)

#### Collection: `notifications`
- **Document ID**: `{notificationId}`
- **Fields**:
  - `title` (`String`): Push notification header
  - `body` (`String`): Message content
  - `category` (`String`): `'Security Alerts'` | `'Vendor Approvals'` | `'Giveaways'` | `'System Maintenance'` | `'Account Updates'`
  - `topic` (`String`, Optional): Target FCM topic
  - `targetEmail` (`String`, Optional): Direct user recipient email
  - `read` (`Boolean`): Delivery/read status
  - `createdAt` (`String`, ISO 8601): Timestamp

## 2. Firebase Storage Directory Structure

All files uploaded through the Android app must use the structured bucket paths:
```
gs://<your-firebase-bucket>/
├── profiles/
│   └── {uid}/
│       └── avatar.jpg
├── marketplace/
│   └── {vendorId}/
│       └── {productId}/
│           ├── item_preview.jpg
│           └── sensitivity_config.json
├── community/
│   └── {postId}/
│       └── banner.jpg
├── verification/
│   └── {vendorId}/
│       └── {applicationId}_id_proof.jpg
└── reports/
    └── {reportId}/
        └── screenshot.jpg
```

## 3. FCM Topic Channels Specification

- `all_users`: Broadcast alerts, system announcements, maintenance windows
- `premium_users`: Exclusive giveaways, VIP sensitivity file drops
- `vendors`: Vendor application status updates, review notifications
- `security_alerts`: Critical security warnings, account access alerts

## 5. Security & Authentication Guidelines for Android

1. **Super Admin Identity**: Keep `ghostfirehub@gmail.com` and `ghostfire@ghost.com` protected in secure Remote Config or Firestore Security Rules.
2. **Serverless Architecture**: Use standard Android Firebase SDKs directly (`FirebaseFirestore.getInstance()`, `FirebaseAuth.getInstance()`). No custom server proxy is required.
3. **App Check Verification**: Enable Firebase App Check with Play Integrity API provider before publishing on Google Play Store.
